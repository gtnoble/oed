# AGENTS.md ‚Äî hed (hed) Coding Agent Guide

## Project Overview

**hed** (`hed`) is a portable port of OpenBSD's `ed(1)` line editor.  Its
goal is to compile cleanly on any minimally POSIX-compliant OS using any C89
compiler.  The codebase is small (~6 500 lines of C) and self-contained: the
OpenBSD regex engine is bundled so no external regex library is required.

Current release: **hed-7.4**  
License: BSD (editor sources), ISC (portability shims)

## Architecture

### Data model
- The editor buffer is a **doubly-linked circular queue** of `line_t` nodes
  (`ed.h`).  Each node stores the line's byte offset and length in a **scratch
  file** (a temporary file opened at startup by `open_sbuf()` in `buf.c`).
  Line text is never kept in memory beyond what is needed for the current
  operation.
- `addr_last` ‚Äî total number of lines; `current_addr` ‚Äî dot (`.`).

### Undo
- Every mutating command pushes `undo_t` records onto a stack (managed in
  `undo.c`).  The `u` command calls `pop_undo_stack()` to replay them in
  reverse.

### Regex
- `re.c` owns the single compiled pattern (`get_compiled_pattern()`).
  The bundled OpenBSD regex library is used instead of the system's to
  guarantee `REG_STARTEND` support (absent from musl and others).  The
  `extended_re` global flag switches between BRE and ERE (`-E` flag).

### Signal handling
- SIGHUP and SIGINT are caught; delivery is deferred by the `SPL1()`/`SPL0()`
  mutex macros (incrementing `mutex`) around critical sections.  Handlers set
  `sighup`/`sigint` flags; `SPL0()` checks and dispatches them.

### Global commands
- `exec_global()` in `glbl.c` builds an active-line list, then repeatedly
  calls `exec_command()` for each matched line.  Moving lines during a global
  command interacts subtly with the active list ‚Äî see the POSIX file for
  documented deviations.

## Build & Test

```sh
# First-time setup (or after adding a portability check to configure)
./configure

# Build
make

# Install (optional)
sudo make install

# Run tests (shell integration + C unit tests)
make test

```

`configure` probes for:
- A working C compiler (`CC` override respected)
- Optional CFLAGS (`-pipe`, `-O2`, `-g`, LTO)
- `pledge(2)` (OpenBSD only)
- `reallocarray`, `strlcat`, `strlcpy` (provided as shims if absent)

`config.h` is the only output consumed by the source; `Makefile` is the other.
Both are `.gitignore`d.

### Test Suite

Tests live under `tests/`.  `make test` builds all binaries and runs
`tests/run_tests.sh`.  Two tiers:

**Tier 1 ‚Äî Shell integration tests** (`tests/sh/t*.sh`)  
Each file sources `tests/harness.sh` and drives the built binary through
stdin, comparing stdout against expected output.  Tests are grouped by
functional area:

| File | Coverage |
|---|---|

| `t02_addresses.sh` | All address forms: `n`, `.`, `$`, `¬±n`, `/re/`, `?re?`, `,`, `;`, `'x`; `=` range form |
| `t03_delete_move_copy.sh` | `d`, `m`, `t` (copy), `j` commands |
| `t04_substitution.sh` | `s` command and all flags (`g`, nth, `p`, `&`, `\1`) |
| `t05_global.sh` | `g`, `v` commands, multi-command global body |
| `t06_file_io.sh` | `e`, `r`, `w`, `W` commands |
| `t07_undo.sh` | `u` command, single and double undo |
| `t08_marks.sh` | `k` command and `'x` addressing |
| `t09_regex.sh` | BRE (default), ERE (`-E`), and PCRE2 (`-P`) |
| `t10_errors.sh` | `?` error token, exit codes, bad addresses |
| `t11_new_commands.sh` | `B` (with undo-depth field), `N`, `y`, `x`, `C` (match count), `K`, `Z` (Adler-32 checksum), `F` (per-line hash / always-hash toggle), `@hash` address form commands; error cases, dot/modified side-effects |
| `t12_flags.sh` | `-l` loose, `-T` transaction (dry-run/wet-run, deferred writes, shell escape ban), `-lT` combined, `-M` agent mode, `-H` always-hash, `-R` read-only mode |
| `t13_shell_escape.sh` | `!cmd` shell escape |
| `t14_pcre.sh` | Comprehensive PCRE2 (`-P`) tests: errors, pattern reuse, syntax features (lookahead/behind, `\w`/`\s`/`\b`/`\d`, non-greedy, named/non-capturing groups, possessive), substitution edge cases (zero-length, nth, `&`), command interactions (`g`, `v`, `?re?`), flag combinations |

**Tier 2 ‚Äî C89 unit tests** (`tests/unit/`)  
`test_utils.c` tests pure utility functions (`has_trailing_escape`,
`strip_escapes`, `translit_text`) by linking against the editor objects.
`main.c` is compiled separately with `-Dmain=_hed_real_main` to avoid a
symbol conflict while still providing all global variables and helper
functions.  The minimal assertion header lives in `tests/test.h`.

**Adding a new test**: add a `run_test` call to the appropriate `sh/t*.sh`
file, or add a new `sh/tNN_name.sh` file (it is picked up automatically by
the glob in `run_tests.sh`).  For a new C unit test binary, add a source
file to `tests/unit/`, add its build rule to `configure` following the
existing `test_utils` pattern, and add the binary name to `TESTBINS`.

## Coding Conventions

- **C89 throughout** ‚Äî no C99/C11 features (`//` comments, VLAs, `_Bool`,
  designated initialisers, `<stdint.h>` types, etc.).  Every new function must
  have a separate forward declaration.
- **`ed.h` is the single shared header** ‚Äî all types, macros, and prototypes
  live there.  Source files include it directly; do not add ad-hoc cross-file
  declarations.
- **Error returns** ‚Äî functions return `0` on success, `ERR` (`-2`) on
  recoverable error, `EMOD` (`-3`) for "buffer modified" prompts, or
  `FATAL` (`-4`) for unrecoverable errors.  Use `seterrmsg()` to set the
  error message before returning `ERR`/`FATAL`.
- **Memory** ‚Äî use the `REALLOC` macro (defined in `ed.h`) for dynamic
  buffers; it handles `SPL1`/`SPL0` around `realloc` and calls `seterrmsg` on
  failure.
- **No external dependencies** ‚Äî do not introduce library calls beyond what
  POSIX guarantees.  If a new function is not universally available, add a
  configure probe and a shim (following the `strlcpy`/`reallocarray` pattern).

## Portability Rules

- The configure script is the **only** portability mechanism.  All
  platform-specific `#ifdef` guards must be driven by macros set in `config.h`
  (never by raw `__linux__`, `__APPLE__`, etc.).
- Shim source files (`reallocarray.c`, `strlcat.c`, `strlcpy.c`) must not be
  compiled on systems that provide the real function ‚Äî the Makefile handles
  this via the object list generated by `configure`.
- Do not use `#pragma` or compiler-specific attributes.

## Common Tasks

### Adding a new command letter
1. Add a `case` branch in the main `switch` in `exec_command()` (`main.c`).
2. Parse addresses with `check_addr_range()` and any command-specific text.
3. Return `0` on success or `ERR` (with `seterrmsg`) on failure.
4. If the command mutates the buffer, push undo records via `push_undo_stack()`
   and set `modified = 1`.
5. Document any POSIX deviation in the `POSIX` file.

### Adding a configure portability probe
1. Write a `foocheck()` shell function in `configure` following the existing
   pattern (compile a test program, set a flag variable).
2. Call it from the main configure flow and emit the appropriate `#define` into
   `config.h`.
3. Add the shim source file if needed; add/omit the `.o` from `OBJS` in the
   generated Makefile.

### Syncing with upstream OpenBSD
- Upstream CVS tags appear in `$OpenBSD:` RCS ID comments at the top of each
  file.  When merging upstream changes, preserve these IDs and update only the
  files that actually changed.
- The bundled regex library (`regcomp.c`, `regexec.c`, `engine.c`,
  `regfree.c`, `regerror.c`, `regex.h`, `regex2.h`, `cclass.h`, `cname.h`)
  tracks `src/lib/libc/regex` in OpenBSD; keep it as a unit.

## AI Agent Usage

### What hed already provides for machine-driven use

**hed-7.4 unified output streams** ‚Äî All `?` error tokens and byte-count
output were moved from `stderr` to `stdout`.  An agent reading only stdout
receives both normal output and error signals on one stream, with no need to
multiplex descriptors.

**Exit codes** ‚Äî hed uses four codes, aligned with GNU ed:

| Code | Meaning |
|---|---|
| `0` | Clean exit |
| `1` | ERR ‚Äî command or address error |
| `2` | EMOD ‚Äî unsaved buffer on quit, or fatal file I/O error |
| `3` | Fatal / internal error |

**Agent-friendly flags** (current `getopt` string: `p:slvETnAMHe:PR`):

| Flag | Effect |
|---|---|
| `-s` | Scripted mode ‚Äî suppresses byte-count output |
| `-l` | Loose mode ‚Äî continue past `ERR`-level errors instead of exiting; combine with `-T` for attempt-all/commit-if-clean semantics |
| `-T` | Transaction mode (dry-run/wet-run) ‚Äî `w`/`W` are deferred; shell escapes forbidden; on any error roll back in-memory and exit 1 without writing; on clean exit execute all deferred writes |
| `-v` | Garrulous mode ‚Äî print explanation for every `?` to stderr |
| `-E` | Use ERE instead of BRE |
| `-P` | Use PCRE2 regular expressions; mutually exclusive with `-E`; requires build with libpcre2-8 |
| `-n` | Always-number ‚Äî prefix every output line with its line number; equivalent to running `N` at startup |
| `-H` | Always-hash ‚Äî prefix every output line with its per-line Adler-32 hash and a tab (`@xxxxxxxx\t`); equivalent to running `F` (toggle) at startup. When combined with `-n`, column order is `linenum\t@hash\tcontent`. Implied by `-M`. |
| `-A` | Success token ‚Äî print `OK <dot>` after every successful command, where `<dot>` is the current line number (e.g. `OK 3`); errors print `?` and do not print `OK` |
| `-e cmd` | Inline command ‚Äî execute `cmd` as a command before reading stdin; repeatable; implies `-s` |
| `-lT` | Combined loose + transaction ‚Äî attempt all commands, commit writes only if every command succeeded; exit 1 and roll back if any error occurred |
| `-M` | Machine/agent mode ‚Äî convenience flag enabling `-s -A -v -l -T -H` simultaneously; does not imply `-n`. **Migration note**: existing `-M` scripts that parse printed line content will now see an extra `@xxxxxxxx\t` column before the content on every printed line. Scripts that only consume `OK N` and `?` tokens are unaffected. |
| `-R` | Read-only mode ‚Äî reject any mutating command (`a`, `c`, `d`, `e`, `E`, `i`, `j`, `m`, `r`, `s`, `t`, `u`, `w`, `W`, `x`, `!`); read and search commands continue to work; distinct from GNU ed's `-r` restricted mode |

**Non-standard commands already implemented:**

| Command | Syntax | Effect |
|---|---|---|
| `B` | `B` | Print buffer state: `current_addr addr_last modified undo_depth filename` ‚Äî `undo_depth` is `1` if undo is available, `0` otherwise |
| `N` | `N` | Toggle `always_number`: prefix every subsequent output line with its line number |
| `y` | `(.,.)y` | Yank addressed lines into the named or unnamed cut buffer (prefix with `"r` to use named register r) |
| `x` | `(.)x` | Put named or unnamed register contents after the addressed line (prefix with `"r` to use named register r) |
| `K` | `K` | List all currently set marks; one line per mark in the form `'x addr` |
| `C` | `(1,$)C/re/` | Count lines in the addressed range matching `re`; print the integer count; current address unchanged; cannot be nested inside a global command |
| `Z` | `(1,$)Z` | Compute Adler-32 checksum of addressed lines; print as 8 lowercase hex digits; each line's bytes are fed with a synthetic `\n` separator so boundary changes are visible; NUL bytes hashed as stored; empty buffer yields `00000001` without error; current address unchanged |
| `=` | `(addr1,addr2)=` | With a range, prints both resolved addresses separated by a space; with a single address or no address, prints that address (existing behaviour) |
| `F` | `F` or `(.,.)F` | Without address: toggle `always_hash` ‚Äî prefix every subsequent printed line with `@xxxxxxxx\t` (per-line Adler-32 hash and tab). With address: print the hash of each addressed line as `@xxxxxxxx`, one per line; current address unchanged. Default address is `.`. Single-line `F` and single-line `Z` produce the same numeric value; `F` adds the `@` sigil. |
| `@xxxxxxxx` | address form | Hash address ‚Äî resolves to the unique line whose per-line Adler-32 equals the 8 hex digits. Error if zero or more than one line matches. Shift-invariant: insertions/deletions elsewhere do not invalidate the address. All blank lines share hash `00020002`; files with more than one blank line cannot use `@00020002`. Use `(.)F` to obtain the hash address of a line. Verify uniqueness before scripting: `C/^exact content$/` must return `1`. |

### Features that will not be implemented

The following have been considered and explicitly rejected as out of scope for
`hed`.  Do not reopen or implement these.

| Feature | Reason |
|---|---|
| JSON output mode (e.g. `-J` flag) | Structured serialisation formats go beyond what a line editor is expected to provide.  Callers that need structured output should post-process `hed`'s existing plain-text output (including the `B` command's machine-readable snapshot) with an external tool such as `jq` or `awk`. |

## Required Practices

Follow these rules on every change.

- **Add tests for every new feature.**  Add `run_test` calls to the
  appropriate `tests/sh/tNN_*.sh` file (or create a new one for a distinct
  functional area).  Tests must be committed in the same change as the code
  and documentation they cover.
- **Update all three documentation files.**  Every change that adds, removes,
  or alters a flag, command, or behaviour must be reflected in `ed.1` (man
  page), `POSIX` (extensions / deviations summary), and `AGENTS.md` (AI agent
  reference), in the same commit.


## What NOT to Do

- Do not run `./configure` unnecessarily ‚Äî it overwrites `Makefile` and
  `config.h`, potentially changing build settings.
- Do not add a test framework without first verifying it builds with all
  supported C89 compilers.
- Do not replace the bundled regex library with the system's unless you first
  audit for `REG_STARTEND` support on every target platform.
- Do not use `getline(3)` (C99/POSIX 2008), `asprintf`, or other non-C89
  stdio extensions.
