# AGENTS.md — OpenED (oed) Coding Agent Guide

## Project Overview

**OpenED** (`oed`) is a portable port of OpenBSD's `ed(1)` line editor.  Its
goal is to compile cleanly on any minimally POSIX-compliant OS using any C89
compiler.  The codebase is small (~6 500 lines of C) and self-contained: the
OpenBSD regex engine is bundled so no external regex library is required.

Current release: **oed-7.4**  
License: BSD (editor sources), ISC (portability shims)

## Architecture

### Data model
- The editor buffer is a **doubly-linked circular queue** of `line_t` nodes
  (`ed.h`).  Each node stores the line's byte offset and length in a **scratch
  file** (a temporary file opened at startup by `open_sbuf()` in `buf.c`).
  Line text is never kept in memory beyond what is needed for the current
  operation.
- `addr_last` — total number of lines; `current_addr` — dot (`.`).

### Undo
- Every mutating command pushes `undo_t` records onto a stack (managed in
  `undo.c`).  The `u` command calls `pop_undo_stack()` to replay them in
  reverse.

### Regex
- `re.c` owns the single compiled pattern (`get_compiled_pattern()`).
  The bundled OpenBSD regex library is used instead of the system's to
  guarantee `REG_STARTEND` support (absent from musl and others).  The
  `extended_re` global flag switches between BRE and ERE (`-E` flag).

### Signal handling
- SIGHUP and SIGINT are caught; delivery is deferred by the `SPL1()`/`SPL0()`
  mutex macros (incrementing `mutex`) around critical sections.  Handlers set
  `sighup`/`sigint` flags; `SPL0()` checks and dispatches them.

### Global commands
- `exec_global()` in `glbl.c` builds an active-line list, then repeatedly
  calls `exec_command()` for each matched line.  Moving lines during a global
  command interacts subtly with the active list — see the POSIX file for
  documented deviations.

## Build & Test

```sh
# First-time setup (or after adding a portability check to configure)
./configure

# Build
make

# Install (optional)
sudo make install

# There is currently no automated test suite (make test prints "No tests")
```

`configure` probes for:
- A working C compiler (`CC` override respected)
- Optional CFLAGS (`-pipe`, `-O2`, `-g`, LTO)
- `pledge(2)` (OpenBSD only)
- `reallocarray`, `strlcat`, `strlcpy` (provided as shims if absent)

`config.h` is the only output consumed by the source; `Makefile` is the other.
Both are `.gitignore`d.

## Coding Conventions

- **C89 throughout** — no C99/C11 features (`//` comments, VLAs, `_Bool`,
  designated initialisers, `<stdint.h>` types, etc.).  Every new function must
  have a separate forward declaration.
- **`ed.h` is the single shared header** — all types, macros, and prototypes
  live there.  Source files include it directly; do not add ad-hoc cross-file
  declarations.
- **Error returns** — functions return `0` on success, `ERR` (`-2`) on
  recoverable error, `EMOD` (`-3`) for "buffer modified" prompts, or
  `FATAL` (`-4`) for unrecoverable errors.  Use `seterrmsg()` to set the
  error message before returning `ERR`/`FATAL`.
- **Memory** — use the `REALLOC` macro (defined in `ed.h`) for dynamic
  buffers; it handles `SPL1`/`SPL0` around `realloc` and calls `seterrmsg` on
  failure.
- **No external dependencies** — do not introduce library calls beyond what
  POSIX guarantees.  If a new function is not universally available, add a
  configure probe and a shim (following the `strlcpy`/`reallocarray` pattern).

## Portability Rules

- The configure script is the **only** portability mechanism.  All
  platform-specific `#ifdef` guards must be driven by macros set in `config.h`
  (never by raw `__linux__`, `__APPLE__`, etc.).
- Shim source files (`reallocarray.c`, `strlcat.c`, `strlcpy.c`) must not be
  compiled on systems that provide the real function — the Makefile handles
  this via the object list generated by `configure`.
- Do not use `#pragma` or compiler-specific attributes.

## Common Tasks

### Adding a new command letter
1. Add a `case` branch in the main `switch` in `exec_command()` (`main.c`).
2. Parse addresses with `check_addr_range()` and any command-specific text.
3. Return `0` on success or `ERR` (with `seterrmsg`) on failure.
4. If the command mutates the buffer, push undo records via `push_undo_stack()`
   and set `modified = 1`.
5. Document any POSIX deviation in the `POSIX` file.

### Adding a configure portability probe
1. Write a `foocheck()` shell function in `configure` following the existing
   pattern (compile a test program, set a flag variable).
2. Call it from the main configure flow and emit the appropriate `#define` into
   `config.h`.
3. Add the shim source file if needed; add/omit the `.o` from `OBJS` in the
   generated Makefile.

### Syncing with upstream OpenBSD
- Upstream CVS tags appear in `$OpenBSD:` RCS ID comments at the top of each
  file.  When merging upstream changes, preserve these IDs and update only the
  files that actually changed.
- The bundled regex library (`regcomp.c`, `regexec.c`, `engine.c`,
  `regfree.c`, `regerror.c`, `regex.h`, `regex2.h`, `cclass.h`, `cname.h`)
  tracks `src/lib/libc/regex` in OpenBSD; keep it as a unit.

## AI Agent Usage

### What oed already provides for machine-driven use

**oed-7.4 unified output streams** — All `?` error tokens and byte-count
output were moved from `stderr` to `stdout`.  An agent reading only stdout
receives both normal output and error signals on one stream, with no need to
multiplex descriptors.

**Exit codes** — oed uses three codes, aligned with GNU ed:

| Code | Meaning |
|---|---|
| `0` | Clean exit |
| `2` | Recoverable error or unsaved-buffer warning |
| `3` | Fatal / internal error |

**Agent-friendly flags** (current `getopt` string: `p:slvET`):

| Flag | Effect |
|---|---|
| `-s` | Scripted mode — suppresses byte-count output |
| `-l` | Loose mode — continue past `ERR`-level errors instead of exiting (continue-on-error) |
| `-T` | Transaction mode — on any `ERR`-level error, roll back all in-memory changes made since startup and exit 2.  Requires that `w` not be called before the failing command; a `w` already executed cannot be undone on disk. |
| `-v` | Garrulous mode — print explanation for every `?` to stderr |
| `-E` | Use ERE instead of BRE |

**Non-standard commands already implemented:**

| Command | Syntax | Effect |
|---|---|---|
| `B` | `B` | Print buffer state: `current_addr addr_last modified filename` on one line — machine-readable snapshot |
| `N` | `N` | Toggle `always_number`: prefix every subsequent output line with its line number |
| `y` | `(.,.)y` | Yank addressed lines into the (single) cut buffer |
| `x` | `(.)x` | Put cut buffer contents after the addressed line |

Note: `always_number` has no corresponding startup flag; use the `N` command
at the top of a script to enable numbered output from the start.

### Features not yet implemented

The following were identified as useful for agents but are absent from the
current codebase.  They are free of conflicts with GNU ed's existing flag and
command namespace:

| Feature | Proposed syntax | Notes |
|---|---|---|
| Inline command string | `-e 'cmd'` flag | Mirrors `sed -e`; avoids temp script files |
| Always-number startup flag | `-n` flag | Wires `always_number=1` at init; `N` command covers runtime use |
| Per-command success token | e.g. `-A` flag | Prints `OK\n` to stdout after every successful command so a pipe-driving agent can detect boundaries |
| Named yank registers | `"a y` / `"a x` | `"` is unused in command position; single cut buffer currently |
| Active marks query | `K` command | No-address command listing all currently set `'a`–`'z` marks |
| Exit code for `ERR` vs `EMOD` | — | Both currently exit `2`; separating them (e.g. `EMOD`→`2`, `ERR`→`1`) would align with GNU ed's full 0/1/2/3 scheme |

## What NOT to Do

- Do not run `./configure` unnecessarily — it overwrites `Makefile` and
  `config.h`, potentially changing build settings.
- Do not add a test framework without first verifying it builds with all
  supported C89 compilers.
- Do not replace the bundled regex library with the system's unless you first
  audit for `REG_STARTEND` support on every target platform.
- Do not use `getline(3)` (C99/POSIX 2008), `asprintf`, or other non-C89
  stdio extensions.
